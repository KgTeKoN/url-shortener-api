## Run server with docker

```bash
$ docker-compose up
```

## Swagger
```
http://localhost:3000/swagger#/
```

## Installation

```bash
$ npm install --legacy-peer-deps
```

## Running the app

```bash
# production mode
$ npm run start:migrate:prod
```

## Test

```bash
# unit tests
$ npm run test
```

## What if this service needed to scale to 10,000 or 100,000 URL generation requests per second?
```
I would recommend using microservices for this purpose. By utilizing microservices, you can 
distribute the workload to each microservice, significantly improving performance. To achieve this, 
you can use Kubernetes, for example, which has an excellent orchestration mechanism. Instead of 
Express, consider using Fastify or Koa - they have significantly higher throughput in terms of 
requests per second.

Store responses, apart from the database, in a cache with a specific time-to-live (TTL). Redis, 
hosted in a Docker container (as implemented in this API), is an excellent choice for this.

Utilize SQL databases (PostgreSQL, MySQL, Oracle Database) that come with optimizations for
handling a large volume of data.

Make use of indexing. Hash indexes are best suited for this API since there's no need for 
sorting or range-based filtering.

Consider using read replicas and write replicas to avoid overloading the primary database.

Please note that these recommendations are provided in the context of optimizing a system for
handling a large volume of data.
```
